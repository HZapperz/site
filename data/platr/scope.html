Can all of this be done in Shopify Liquid? Yes. My Fit Foods achieved their entire design in a customized Dawn theme — we verified it. But their site loads 102 scripts on every page and pays for enterprise session recording (Contentsquare). Every feature added to Liquid compounds the performance hit with no code splitting or lazy loading. The UI patches above are worth doing now — but beyond these, we strongly recommend the custom build. It's genuinely faster to develop in React than to hack into Liquid, the site performs better out of the box, and it unlocks capabilities that are simply not possible within Shopify's theme system.

The key difference from patching Liquid: In a custom Next.js build, we are not bound by Shopify's app ecosystem. Zapiet's delivery picker? We build our own — cleaner UX, zero monthly fee. Loox review widget? We pull their API and render it exactly how we want. Cart countdown timer? 20 lines of React. Looping through apps and integrations that fight each other is replaced by owned code that does exactly what it's supposed to do.
Why not keep adding JS to Shopify? Every script injected into a Liquid theme loads on every page — no code splitting, no tree shaking, no lazy loading. Zapiet, Loox, Klaviyo, and PageFly already inject their own scripts that can conflict with each other. The more JS you add in Phase 1 (AJAX cart, countdown timers, Zapiet restyling, checkout extensions), the slower the site gets — you're fighting the platform. In Next.js, code splitting is automatic: each page loads only what it needs. Features that take hours to hack into Liquid take minutes in React. And there's no ceiling — session recording, A/B testing, multi-store architecture, referral programs, real-time analytics dashboards. It's all just code you own, running on infrastructure you control, scaling without per-feature app costs.

Why build custom instead of buying SaaS? Pre-AI, buying 20 off-the-shelf tools and stitching them together was the only economically sensible path. That math has flipped. Building custom now costs the same or less — and gives full ownership, zero vendor lock-in, and a compounding advantage as the business scales. Platr currently pays ~$95–185/mo in app subscriptions. At scale, that number grows fast (Klaviyo alone hits $1,700/mo at 50K contacts). A custom platform hosted on Vercel at ~$20/mo eliminates most of that permanently.
