Can all of this be done in Shopify Liquid?
Yes. My Fit Foods achieved their entire design in a customized Dawn theme — we verified it. But their site loads 102 scripts on every page and pays for enterprise session recording (Contentsquare). Every feature added to Liquid compounds the performance hit. The UI patches above are worth doing now — but beyond these, we strongly recommend the custom build. It’s genuinely faster to develop in React than to hack into Liquid, the site performs better with automatic code splitting, and it unlocks capabilities that are simply not possible within Shopify’s theme system.

Why not keep patching Shopify? Every JS script injected into a Liquid theme loads on every page — no code splitting, no lazy loading. Zapiet, Loox, Klaviyo, and PageFly already inject their own scripts; adding AJAX cart, timers, and checkout extensions on top makes the site slower, not faster. In Next.js, code splitting is automatic — each page loads only what it needs. Features that take hours to hack into Liquid take minutes in React. And there’s no ceiling: session recording, A/B testing, multi-store, referral programs, real-time dashboards — it’s all just code you own.

  • Performance + infrastructure — Automatic code splitting (faster than Liquid), mobile-first responsive design, multi-store ready architecture, SEO optimized
